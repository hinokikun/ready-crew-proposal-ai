# ProposalPilot サポートポリシー

対象製品: ProposalPilot / AI営業秘書
提供者: 要入力
適用開始日: 要入力

## 1. サポート対象

サポート対象:

- ログイン
- ユーザー管理
- 案件入力
- AI提案書生成
- 提出前チェック
- PowerPoint / PDF出力
- Beautiful.ai連携
- 作成履歴
- 監査ログ
- Organization / Workspace
- 管理画面

サポート対象外:

- 利用者端末固有のOS障害
- 利用者ネットワーク障害
- 外部サービス側の仕様変更による一時的な制限
- 利用者が入力した実データの正確性判断
- AI生成結果をそのまま社外提出したことによる業務判断

## 2. 問い合わせ窓口

| 項目 | 内容 |
| --- | --- |
| サポート窓口 | 要入力 |
| 受付方法 | 要入力 |
| 受付時間 | 要入力 |
| 緊急連絡先 | 要入力 |
| 対応言語 | 日本語 |

問い合わせ時に共有してほしい情報:

- 発生日時
- 操作内容
- 期待した結果
- 実際の結果
- Role
- Organization
- Workspace
- request_id
- 画面幅
- ブラウザ
- スクリーンショット名

共有しないでほしい情報:

- Password
- APIキー
- Token
- Authorization
- 実顧客の機密本文
- 個人情報

## 3. 障害重要度

| 重要度 | 内容 | 初回応答目安 | 復旧目標 |
| --- | --- | --- | --- |
| Critical | 全利用者がログイン不可、DB接続不能、重大な権限漏れ | 要入力 | 要入力 |
| High | 主要機能が利用不可、PPTX / PDF / Beautiful.aiが広範囲で失敗 | 要入力 | 要入力 |
| Medium | 一部利用者または一部機能の不具合 | 要入力 | 要入力 |
| Low | 表示崩れ、文言、軽微な操作性改善 | 要入力 | 要入力 |

## 4. 障害時の案内方針

- 利用者には分かりやすい日本語で案内する
- 技術的な詳細は管理者向けに分ける
- 原因未確定の場合は断定しない
- 暫定回避策がある場合は明記する
- 復旧後に再発防止策を記録する

## 5. サポートの優先順位

1. セキュリティと権限
2. ログインと認証
3. 提案書作成と出力
4. Beautiful.ai連携
5. 管理者操作
6. 表示・文言・軽微な改善

## 6. 対応ブラウザ

| ブラウザ | 対応 |
| --- | --- |
| Google Chrome 最新版 | 対応 |
| Microsoft Edge 最新版 | 対応 |
| Safari 最新版 | 要確認 |
| Firefox 最新版 | 要確認 |

スマートフォン表示は、360px以上の画面幅を基準に確認します。

## 7. 変更・要望の扱い

機能追加要望は、障害対応とは分けて管理します。

- 業務影響
- 利用頻度
- 対象Role
- Organization / Workspaceへの影響
- セキュリティ影響
- 実装優先度

を確認し、次期Version候補として整理します。

## 8. サポート履歴

サポート履歴には、秘密情報や顧客本文全文を保存しません。

保存する内容:

- 問い合わせ種別
- 発生日時
- 影響範囲
- 対応状況
- request_id
- 解決内容
- 再発防止
