# ログインガイド

## ログイン入口

AI営業秘書のログイン画面には、次の2つの入口があります。

| 入口 | 対象 | 主な用途 |
| --- | --- | --- |
| 利用者ログイン | 一般利用者、閲覧者 | 案件入力、AI Workspace、提案書作成、PPT/PDF、Beautiful.ai、自分の案件確認 |
| 管理者ログイン | 管理者、組織管理者 | ユーザー管理、Organization / Workspace管理、Analytics、監査ログ、Prompt Studio、Knowledge管理、Learning、Pilot、Integrations、Release、Maintenance、UAT |

## 対象ロール

| 表示名 | 内部ロール |
| --- | --- |
| 管理者 | `admin` |
| 組織管理者 | `manager` |
| 一般利用者 | `member`、互換入力 `user` |
| 閲覧者 | `viewer` |

`manager` は既存仕様との互換のため、管理者ログイン側で扱います。通常の新規ユーザー作成画面では、管理者・一般利用者・閲覧者の3択を使います。

## 権限不一致時の案内

- admin / manager が利用者ログインを選ぶと、管理者ログインからログインするよう案内します。
- member / viewer が管理者ログインを選ぶと、管理者権限がないことを案内します。
- `login_mode` 未指定の既存クライアントは従来どおりログインできます。

## パスワード忘れ時の運用

1. 利用者は管理者へ連絡します。
2. 管理者がユーザー管理画面で一時パスワードを再設定します。
3. 利用者は再設定されたパスワードで該当する入口からログインします。

## セキュリティ注意

- パスワードはブラウザに保存しません。
- ログアウト後に保存されるのは、次回表示用の `login_mode` のみです。
- 監査ログにはパスワード、トークン、APIキー、Authorizationヘッダー、メールアドレス全文を保存しません。
- 管理者ログインURLや `?login=admin` を知っていても、Backendのロール判定を通らない限り管理者にはなれません。
