# Role Permissions

## Version 24.0 正式運用ロール

| Role | 権限範囲 | 主な許可 | 主な禁止 |
| --- | --- | --- | --- |
| `admin` | 全Organization / Workspaceの管理範囲 | ユーザー管理、監査ログ、診断、UAT、Organization / Workspace管理、全業務機能 | 最後の有効adminの無効化・削除 |
| `manager` | 所属Organization / Workspaceの管理範囲 | 案件、提案書、レビュー、作成履歴、運用状況の閲覧と管理 | ユーザー作成・削除、秘密設定閲覧、システム診断設定変更 |
| `member` / `user`互換 | 所属Workspaceの通常利用範囲 | 案件作成、AI生成、PPTX/PDF/Beautiful.ai出力、自分または許可範囲の履歴閲覧 | 管理機能、ユーザー管理、監査ログ閲覧 |
| `viewer` | 所属Workspaceの閲覧範囲 | 案件、履歴、状態の閲覧 | 案件作成、AI生成、出力、編集、承認、管理操作 |

- APIで `role: "user"` を受け取った場合は、後方互換のため内部保存時に `member` へ正規化します。
- 管理者機能はFrontendの非表示だけでなく、Backendの `require_roles` で必ず拒否します。
- ロール変更、無効化、パスワード変更、論理削除では `auth_version` を更新し、古いTokenを拒否します。
- `admin` 自身の無効化・削除、および最後の有効adminの無効化・削除は禁止します。

## Version 23.1 ログイン入口と表示名

| ログイン入口 | 許可ロール | 表示名 |
| --- | --- | --- |
| 利用者ログイン | `member`, `user`互換, `viewer` | 一般利用者 / 閲覧者 |
| 管理者ログイン | `admin`, `manager` | 管理者 / 組織管理者 |

- `admin` が利用者ログインを選んだ場合は、管理者ログインからログインするよう案内します。
- `member` / `viewer` が管理者ログインを選んだ場合は、管理者権限がないことを案内します。
- `manager` は既存のレビュー・公開承認・一部運用管理との互換のため、管理者ログイン側で扱います。
- `login_mode` が未指定の既存クライアントは従来互換として入口判定を行いません。

AI営業秘書の画面上のロールは、社内利用者に分かりやすいように次の2区分へ整理します。

| 表示ロール | 内部互換ロール | 説明 |
| --- | --- | --- |
| 管理者 | `admin` | すべての運用・管理機能を利用できます。 |
| 一般利用者 | `member`, `user`入力 | 案件入力、AI Workspace、提案書作成、PPT/PDF、Beautiful.ai作成、自分の案件確認を利用できます。 |
| 一般利用者（閲覧のみ） | `viewer` | 既存データ互換のため一時的に維持します。生成・編集・承認はできません。 |
| 管理者（互換） | `manager` | 既存データ互換のため維持します。レビュー承認や一部運用機能で管理者互換として扱います。新規作成UIでは選択しません。 |

## 既存ロールの扱い

- `admin` はそのまま管理者です。
- `member` は画面上「一般利用者」と表示します。
- APIで `role: "user"` を受け取った場合、既存DB制約との互換性のため内部保存は `member` に正規化します。
- `viewer` は当面ログイン互換のため残しますが、生成・編集・承認はできません。
- `manager` は既存テストと運用互換のため一部管理機能で管理者互換として残します。通常のユーザー作成画面には出しません。

## セキュリティ運用メモ

- 管理者操作はFrontendで隠すだけでなく、Backendの `require_roles` で強制します。
- ユーザー無効化、ロール変更、パスワード再設定、Pilot対象変更は `auth_version` を更新し、古いログイントークンを無効化します。
- 最後の管理者を無効化、または一般利用者へ変更することはできません。
- 監査ログにはパスワード、トークン、APIキー、顧客本文、生成本文を保存しません。
## Version 23.0 通常モードと詳細モード

- `member` / `user` / `viewer` は通常モードを初期表示します。
- 通常モードでは管理者向け診断、Prompt Studio、Learning、Queue Monitor、Audit Log、Integration内部設定を表示しません。
- `admin` / `manager` は詳細モードをONにすると管理者向け診断、UAT、管理者メニューを確認できます。
- Backendの権限仕様は変更していません。Frontendの表示整理だけで管理者権限を付与しません。
