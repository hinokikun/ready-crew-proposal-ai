# Proposal Improvement Backlog

The Proposal Improvement Backlog stores safe improvement candidates generated by the Proposal Optimization Engine.

## Stored Fields

- `priority`
- `impact`
- `confidence`
- `expected_improvement`
- `effort`
- `importance`
- `adoption_rate`
- `predicted_win_rate_delta`
- `composite_score`
- `status`
- `owner`
- `explanation`
- `simulation_json`
- `organization_id`
- `workspace_id`

## Status

- `suggested`: candidate is available.
- `selected`: a user selected the recommendation.
- `approved`: manager/admin approved it.
- `in_revision`: selected for Presentation Revision work.
- `applied`: reflected in a revision or related proposal output.
- `measured`: outcome was measured from safe aggregate data.
- `rejected`: rejected by operations.
- `archived`: no longer active.

Invalid status transitions are rejected by the backend with HTTP 409.

Legacy values are mapped as follows:

- `open` -> `suggested`
- `adopted` -> `selected`
- `done` -> `applied`
- `deferred` -> `archived`

## Priority Calculation

Priority is based on:

- Expected win-rate improvement
- Effort
- Importance
- Historical adoption rate
- Confidence

The composite score is intentionally simple and explainable. It is not LLM fine-tuning.

Default formula:

```text
optimization_score =
impact * 0.30
+ confidence * 0.25
+ urgency * 0.20
+ historical_adoption * 0.15
- effort * 0.10
```

Weights can be configured with:

- `OPTIMIZATION_WEIGHT_IMPACT`
- `OPTIMIZATION_WEIGHT_CONFIDENCE`
- `OPTIMIZATION_WEIGHT_URGENCY`
- `OPTIMIZATION_WEIGHT_ADOPTION`
- `OPTIMIZATION_WEIGHT_EFFORT`

## Evidence Governance

Each backlog item stores:

- `evidence_type`
- `sample_size`
- `is_estimate`
- `calculation_method`
- `predicted_effect_json`
- `measured_effect_json`
- `measurement_status`
- `requires_human_review`
- `evidence_summary`
- `evidence_period`

Predicted effects are reference estimates. They are not guaranteed results.

If `sample_size` is below `OPTIMIZATION_MIN_SAMPLE_SIZE`, the system lowers confidence and requires human review.

## Data Safety

Do not store:

- Full proposal text
- Customer emails
- Customer confidential data
- API keys
- Passwords
- Beautiful.ai tokens
- Raw Beautiful.ai responses

## Operations

Recommended workflow:

1. Run optimization after Presentation Review.
2. Review TOP5 recommendations.
3. Adopt relevant items.
4. Create a Presentation Revision from selected review actions.
5. Manager/admin approves high-impact recommendations.
6. Track adoption and revision success in Analytics.
