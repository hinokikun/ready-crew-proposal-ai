目的: 販促物で使用する表現を、根拠のある事実・条件付き表現・使用禁止表現に分け、制作時の誤認や過度な訴求を防ぐ。

## 1. 製品情報の正確性監査

### 本番実機確認済みとして扱える機能

以下は、Version 24.0までの確認記録および本番動作確認の報告に基づき、販促物で「利用できる機能」として記載できます。

| 機能 | 販促での表現 | 注意 |
|---|---|---|
| 案件情報入力 | 案件メールやヒアリングメモを入力できます | 実顧客情報の画面掲載は禁止 |
| AI案件分析 | 入力内容を整理し、提案作成を支援します | AIの判断は人が確認する前提 |
| 提案ストーリー生成 | 提案の流れづくりを支援します | 成果を保証しない |
| 提出前チェック / Quality Gate | 社外提出前の確認項目を通せます | 法務確認の代替とは書かない |
| 要約PowerPoint生成 | 要約版PowerPointを出力できます | PowerPoint品質はUATで継続確認 |
| 詳細PowerPoint生成 | 詳細版PowerPointを出力できます | デザイン調整が必要な場合あり |
| 見積PDF生成 | 見積PDFを出力できます | 金額・条件は人が最終確認 |
| Beautiful.ai生成 | Beautiful.aiでプレゼンを生成できます | Beautiful.ai側設定とAPI状態に依存 |
| Beautiful.ai診断 | 管理者が連携状態を確認できます | APIキーは表示しない |
| 管理者機能 | ユーザー、権限、診断、監査を管理できます | 管理者向け機能として記載 |

## 2. 使用可能な事実

以下は、販促物で使用可能です。

- ProposalPilotは、AI営業秘書として提案書作成業務を支援する製品です。
- 案件情報入力から、提案内容整理、PowerPoint、PDF、Beautiful.ai出力までを支援します。
- 提出前チェックにより、社外提出前の確認プロセスを画面上で支援します。
- 管理者はユーザー管理、権限管理、監査ログ、Beautiful.ai診断を確認できます。
- Organization / Workspace単位の利用管理を前提としています。
- AI生成結果は人が確認してから提出する運用を前提としています。

## 3. 条件付きで使用可能な表現

| 表現 | 使用条件 | 推奨補足 |
|---|---|---|
| 提案資料作成の負担を軽減 | 実測値なしで使用可 | 効果は利用環境により異なる |
| 作成時間の短縮を支援 | 実測値なしで使用可 | 削減率は書かない |
| 提案品質の標準化を支援 | チェック機能とテンプレート運用を説明する場合 | 品質保証とは書かない |
| 案件入力から各種出力まで一気通貫で支援 | 対応出力を明記する場合 | 外部連携を含めない |
| 営業担当が迷わず進めやすいUI | ガイドUIを説明する場合 | すべての利用者が迷わないとは書かない |
| 管理者が利用状況を確認しやすい | 管理画面説明の範囲 | 高度なBIとは書かない |

## 4. 使用禁止または要検証の表現

以下は、根拠が揃うまで確定コピーに使用しません。

- 作業時間90%削減
- 10分で完成
- 受注率向上
- 売上向上
- 誰でも高品質
- 必ず成果が出る
- 自動で受注できる
- 人の確認が不要
- 法務確認を自動化
- 競合製品より必ず優れている
- Beautiful.aiで必ず美しい資料になる
- すべての営業資料を完全自動化

## 5. 今後計測すべきKPI

- 案件入力から要約PowerPoint出力までの平均時間
- 案件入力から詳細PowerPoint出力までの平均時間
- 見積PDF出力までの平均時間
- Quality Gate完了率
- Beautiful.ai生成成功率
- AIレビュー実行率
- 改善候補の採用率
- 作成履歴からの再利用率
- 利用者の満足度
- 管理者によるエラー確認時間
- 初回利用者の完了率
- UATにおける重大不具合数

## 6. 導入効果の測定方法

1. 導入前に、提案資料作成にかかる平均時間を記録する。
2. 導入後に、同条件の案件で入力から成果物出力までの時間を記録する。
3. PowerPoint、PDF、Beautiful.aiの生成成功率を月次で確認する。
4. Quality Gateで未確認になりやすい項目を集計する。
5. 営業担当と管理者に、月1回の簡易アンケートを実施する。
6. 効果表現は、最低でも10件以上の実測データが揃ってから数値化する。

## 7. 制作時の注意

- 未確認の効果数値を大きく表示しない。
- 実顧客名、実メール、実電話番号、APIキー、request_idを画像に載せない。
- 競合比較は公開情報で確認できる一般的な機能範囲に限定する。
- 「AIが判断する」ではなく「AIが支援し、人が確認する」と表現する。
- 料金、会社名、問い合わせ先、導入実績は未確定の場合「要入力」とする。
